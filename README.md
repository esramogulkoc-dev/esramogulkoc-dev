#### 🤖 AI Agents/ RAG/ Apps

🔗 [**Retail Sales Chatbot**](https://github.com/esramogulkoc-dev/Retail_sales_chatbot) — Retails — Natural Language to SQL

- 🛍️ **Overview:** A retail Q&A system that lets users query a t-shirt inventory database using plain English.  
- 🧠 **NL to SQL:** Converts natural language questions into SQL queries.  
- 📈 **Execution & Output:** Executes queries and returns human-readable answers.

🔗 [**BeCode Rules — RAG Chatbot**](https://github.com/esramogulkoc-dev/rag_doc_qa) — A RAG-based AI chatbot that answers questions about BeCode bootcamp rules using only the official document as ground truth.

🧠 **RAG Pipeline:** LangChain (TextLoader, RetrievalQA), FAISS, Azure OpenAI (text-embedding-3-small, gpt-4.1-mini)  
✂️ **Chunking:** RecursiveCharacterTextSplitter (500 / 100 overlap)  
💬 **UI:** Streamlit chat interface with source display + reset option  
🚫 **Anti-hallucination:** Explicit refusal when context is missing  
🧪 **Testing:** Notebook with valid + “trap” queries to validate grounding  
☁️ **Deploy:** Azure App Service (Linux, Python 3.11, startup.sh, env vars)  

🔗 [**Azure AI Foundry Agent — **BeCode Rules — RAG Chatbot**](https://github.com/esramogulkoc-dev/azuredev-f4d4/tree/main) — Azure AI Foundry — No-code RAG Agent

- ☁️ **Overview:** A fully no-code RAG chatbot built directly in Azure Portal using Azure AI Foundry with a custom knowledge base.  
- 📄 **RAG Pipeline:** Documents → chunking → embeddings → vector search → GPT response generation  
- 🧠 **LLM Integration:** Azure OpenAI model managed via Foundry orchestration  
- 🔎 **Retrieval:** Top-k relevant chunks automatically injected into prompts for grounded answers  
- 🚫 **Guardrails:** Responses strictly limited to uploaded data (no hallucination)  
- ⚙️ **Setup:** Entirely configured via Azure Portal UI (agent, data sources, instructions)  
- 🚀 **Deployment:** One-click deployment as a hosted AI agent in Azure Foundry
